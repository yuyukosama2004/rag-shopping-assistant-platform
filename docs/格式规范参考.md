# 淮阴工学院毕业设计（论文）格式规范参考

> 基于两篇往届毕业论文（田彬-智慧养老、戴钰蓉-药械采购）提取的格式规范。

---

## 一、论文整体结构

两份论文均采用以下章节组织：

```
封面
中文摘要
外文摘要（英文）
目录
1 引言
  1.1 研究背景及意义
  1.2 国内外研究现状
  1.3 主要工作 / 论文结构
2 相关理论与技术（或 相关技术）
3 系统分析
  3.1 可行性分析
  3.2 功能需求分析
  3.3 非功能需求分析
  3.4 UML用例图 / 系统流程图
4 系统设计
  4.1 系统架构设计（含架构图）
  4.2 数据库设计（E-R图 + 数据表结构）
  4.3 功能模块详细设计
5 系统实现
  （核心功能模块的实现描述 + 界面截图）
6 系统测试
  （测试用例 + 测试结果）
7 总结与展望
致谢
参考文献
```

**对照我们的项目章节建议：**
```
1 引言
  1.1 研究背景及意义
  1.2 国内外研究现状（微服务电商架构演进、AI导购技术）
  1.3 主要工作
2 相关理论与技术
  Spring Cloud Alibaba微服务、Vue.js、Redis、RabbitMQ、Sentinel、大语言模型
3 系统分析
  3.1 可行性分析
  3.2 功能需求分析（用例图）
  3.3 非功能需求分析（高并发、可用性）
4 系统设计
  4.1 系统架构设计（微服务分层架构图）
  4.2 数据库设计（E-R图 + 数据表）
  4.3 核心模块详细设计（高并发下单链路、AI导购链路）
5 系统实现
  5.1 微服务基础架构实现
  5.2 高并发下单模块实现
  5.3 AI导购模块实现
  5.4 流量防护与限流实现
6 系统测试
7 总结与展望
```

             （姓名） （专业技术职务）
校外指导老师： /    /
             （姓名） （专业技术职务）

- 校外指导老师若无则填 /

## 三、页眉页脚

**页眉：** 每页上方居中或左对齐显示 `淮阴工学院毕业设计（论文）`

**页脚：** 页码格式为 `第 X 页 共 Y 页`

**目录页：** 页码使用罗马数字 `第 I 页 共 II 页`

## 四、摘要格式

```
                    毕业设计（论文）中文摘要

[正文：200-400字左右，概述研究背景、技术方案、完成内容、结果]

关键词 XXX，XXX，XXX（3-5个关键词，分号或逗号分隔）
```

### 英文摘要
```
                    毕业设计（论文）外文摘要

Title: English Title

Abstract:
[英文论文摘要]

Keywords XXX, XXX, XXX
```

## 五、章节标题层级

```
1 引言 ........................................ 一级标题（黑体/加粗，较大字号）
1.1 研究背景及意义 .......................... 二级标题
1.1.1 XXX ................................... 三级标题（可选，两份论文都有用到）
```

正文段落：首行缩进两字符，两端对齐。

## 六、图表格式

**图：** 图序 + 图名，位于图下方居中
```
图 1 系统架构图
图 2 高并发下单业务流程图
```

**表：** 表序 + 表名，位于表格上方
```
表 1 用户表（t_user）
表 2 订单状态枚举表
```

## 七、参考文献格式

采用 **GB/T 7714** 标准，`[序号]` 标注：

**中文期刊：**
```
[1] 作者. 题名[J]. 刊名, 年份, 卷(期): 起止页码.
例如：王璐, 姜宇轩, 李青山, 等. 微服务故障检测研究综述[J]. 计算机学报, 2023, 46(11): 2342-2369.
```

**英文期刊：**
```
[2] Author A, Author B. Title[J]. Journal, Year, Vol(Issue): Pages.
例如：Quattrocchi G, Cocco D, Staffa S, et al. Cromlech: Semi-automated monolith
     decomposition into microservices[J]. IEEE Transactions on Services Computing,
     2024, 17(2): 640-653.
```

**图书：**
```
[3] 作者. 书名[M]. 版本. 出版地: 出版社, 年份.
例如：Newman S. Building Microservices[M]. 2nd ed. O'Reilly Media, 2021.
```

**在线文献：**
```
[4] 作者. 题名[J/OL]. 刊名, 1-20[引用日期]. 网址.
例如：李志, 夏书婷, 李圣杰, 等. 容器文件系统隔离增强机制[J/OL]. 软件学报, 1-20[2026-01-22].
```

## 八、排版要求（推测，基于PDF呈现效果）

| 元素 | 字号/字体推测 |
|------|-------------|
| 论文标题（封面） | 二号或小二号黑体 |
| 学校名（封面） | 小初号或一号黑体 |
| 一级标题（1 引言） | 三号或小三黑体加粗 |
| 二级标题（1.1） | 四号黑体加粗 |
| 三级标题（1.1.1） | 小四黑体 |
| 正文 | 小四宋体，1.5倍行距 |
| 页眉 | 五号宋体 |
| 页脚 | 五号宋体 |
| 摘要正文 | 小四宋体 |
| 参考文献 | 五号或小五宋体 |
| 表格内容 | 五号宋体 |

**注意：** 以上字号为根据 PDF 呈现效果的推测。正式论文应使用学校官方模板。

## 九、关键写作要点

1. **引用规范：** 开题报告的参考文献可以直接复用，都是往届学生用过的格式
2. **章节平衡：** 每个一级章节下至少有两个二级章节（如 1.1、1.2），不能只写一个
3. **系统截图：** 实现章节必须配系统运行截图（前端界面、API文档、数据库表结构等）
4. **测试章节：** 需要写测试用例表格 + 测试结果分析
5. **英文摘要：** Title 单独一行，Abstract 用英文写一段连续文字
6. **致谢：** 感谢导师、同学、家人，约300-500字
