# 确定性库存洞察

## 1. 目标

`INV-02` 在 `order-service` 提供只读、可重复计算的 SKU 库存经营事实，不依赖 AI Key，也不调用
大模型。接口只面向店主和店员。

本阶段提供：

- 库存洞察列表、风险筛选和稳定排序。
- 各风险数量与库存汇总。
- 单 SKU 的计算事实、最近确认销售摘要和库存流水证据。
- 固定时区、时间窗口、金额精度、权限和隐私测试。

## 2. 非目标

- 不包含商家前端页面或 AI 解释。
- 不自动修改价格、库存、商品状态、订单或退款。
- 不在缺少采购成本和采购批次时计算利润、资金占用或精确库龄。
- 不按退款记录推断 SKU 退款数量；响应会明确披露这一限制。

## 3. 业务口径

- 目标时区固定为 `Asia/Shanghai`。
- 确认销售以 `t_order.pay_time IS NOT NULL` 为准，数量来自
  `t_order_item.quantity`，金额来自 `t_order_item.subtotal`。
- 需求量使用订单创建时间，排除 `CANCELLED(4)` 和 `TIMEOUT(5)`。
- COD 未确认收款可计入需求量，但不计入确认销量和确认收入。
- 7/30/90 天窗口均包含起点和计算时刻；一次请求中的所有 SKU 共用同一个计算时刻。
- `dailyVelocity30d = confirmedQty30d / 30`，保留 4 位小数。
- `daysOfCover = available * 30 / confirmedQty30d`，保留 1 位小数；30 天确认销量为 0 时返回
  `null`。
- `sellThrough90d = confirmedQty90d / (confirmedQty90d + available)`，保留 4 位小数；
  分母为 0 时返回 `null`。
- 确认收入保留 2 位小数。

风险按下列顺序命中第一条：

1. `OUT_OF_STOCK`：销售中且 `available = 0`。
2. `LOW_STOCK`：销售中、`0 < available <= 5` 且近 30 天有确认销量。
3. `DEAD_STOCK`：`available > 0`、SKU 至少 30 天且近 90 天无确认销量。
4. `SLOW_MOVING`：`available > 0`、近 30 天无确认销量、近 90 天有确认销量。
5. `OVERSTOCK`：`available >= 10` 且 `daysOfCover > 90`。
6. `HEALTHY`：未命中上述规则。

默认排序为风险优先级、`productId`、`skuId`，保证相同输入下稳定。

## 4. API

```text
GET /api/merchant/ai/inventory-insights
GET /api/merchant/ai/inventory-insights/summary
GET /api/merchant/ai/inventory-insights/{skuId}/evidence
```

列表参数：

```text
pageNum=1
pageSize=20
keyword=
risk=DEAD_STOCK
sort=RISK_DESC
```

- `pageSize` 范围为 1–100。
- `keyword` 最长 100 个字符，匹配商品名或 SKU 编码。
- 当前唯一稳定排序值为 `RISK_DESC`。
- 接口只接受网关注入的 `X-User-Role` 中 `OWNER(1)` 或 `STAFF(2)`。

证据响应不会包含消费者姓名、手机号、地址、Token 或商家内部订单备注。最近确认销售仅返回收款时间、
数量和小计；库存流水返回库存快照与业务引用。

## 5. 配置

```yaml
inventory:
  insight:
    low-stock-threshold: 5
    dead-stock-min-age-days: 30
    dead-stock-window-days: 90
    overstock-min-available: 10
    overstock-cover-days: 90
    time-zone: Asia/Shanghai
```

阈值变更必须同步更新人工手算夹具、自动化测试和后续评测基线。

## 6. 查询计划

聚合查询复用现有索引：

- `t_order_item.idx_order_item_sku`
- `t_order.PRIMARY`
- `t_stock.uk_stock_sku`

Testcontainers MySQL 测试执行 `EXPLAIN` 并确认 SKU 到订单明细、订单主键的候选索引可用。当前没有证据
支持新增重复索引，因此 `INV-02` 不增加 Flyway 迁移。
