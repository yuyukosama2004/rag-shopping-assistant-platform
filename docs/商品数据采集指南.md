# 商品数据自动采集指南

## 做什么

从 中关村在线(ZOL) 自动爬取每款手机的：图片、价格、规格参数、外观颜色、存储配置，并写入 MySQL。

## 安装依赖

```bash
pip3 install requests beautifulsoup4 mysql-connector-python
```

## 确认图片目录

```
biyesheji-frontend/
└── public/
    └── images/        ← 图片会下载到这里
        1001.jpg
        1002.jpg
        ...
```

如果 `public/images/` 不存在，脚本会自动创建。

## 运行

```bash
cd /home/jill/project/biyesheji
python3 docs/fetch_products.py
```

## 可能遇到的情况

| 情况 | 处理 |
|------|------|
| "未找到 ZOL 页面" | 该机型 ZOL 没有或不匹配，手动找图 |
| "图片下载失败" | 图片链接过期或防盗链，手动下载 |
| 价格解析不准 | ZOL 显示的是电商均价，以实际为准 |
| 规格不全 | 某些机型 ZOL 参数页不完整 |
| 被 ZOL 限流 | 脚本已设 2 秒间隔，如仍被限可调到 5 秒 |

## 执行后

1. 检查图片: `ls biyesheji-frontend/public/images/` 看是否有 50 张
2. 检查数据库: `SELECT id, name, main_image, color_options, storage_options FROM t_product WHERE status=1;`
3. 如果有缺失，手动补图
4. 重新构建前端: `npm run build`
5. 重启 order-service 重建 RAG 向量
