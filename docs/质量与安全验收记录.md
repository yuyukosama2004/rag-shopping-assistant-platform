## 2. 已验证的不变量

- 客户端伪造商家角色头不能获得商家权限。
- Refresh Token 不能作为 Access Token 调用业务接口。
- 同一订单的重复接单、发货操作不会产生第二次状态跃迁。
- 并发购买成功量不超过可售库存，数据库库存不为负数。
- MySQL、Redis 或单个业务服务恢复后，健康检查和业务路由能够恢复。
- 生产部署中的应用、数据库、缓存和监控端口不直接暴露到公网。

## 4. 尚未完成的现场验收

- 真实域名、公开 DNS、Let's Encrypt 签发、自动续期和公网 80/443 暴露面检查。
- 至少一家真实商家在无开发人员代操作的情况下完成商品导入、上架、下单、发货和售后演练。
- 外部告警接收人（邮件、Webhook 或企业协作工具）在真实故障中的送达确认。
- 真实支付不在当前版本范围；当前只适合 `COD/OFFLINE` 流程。
- S3 兼容代码已验证，但具体对象存储供应商仍需执行权限、网络、备份和恢复现场验收。

在上述事项完成前，发布标识应保持预发布版本，不得标记为稳定生产版。
