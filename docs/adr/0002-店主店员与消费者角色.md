# ADR-0002：店主、店员与消费者角色

- 状态：已采纳
- 日期：2026-07-15

## 背景

商家需要安全地管理商品和订单，同时消费者注册不能获得后台能力。单纯依赖前端隐藏菜单不能形成安全边界。

## 决策

采用 `OWNER`、`STAFF`、`CUSTOMER` 三类稳定角色。首个 `OWNER` 只能通过一次性初始化令牌创建；普通注册只能创建 `CUSTOMER`。网关负责商家路由的首层鉴权，各业务服务继续校验角色。`OWNER` 管理店铺和店员，`STAFF` 只执行授权的日常运营，关键操作进入审计记录。

## 影响

- 不提供默认店主密码，也不从消费者账户“升级”为店主。
- 新增后台接口必须同时具备网关保护、服务层角色校验和必要的审计。
- 若日后需要细粒度权限，应在现有三类角色上增加权限集合，不改变消费者的隔离边界。
