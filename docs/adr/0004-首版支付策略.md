# ADR-0004：首版支付策略

- 状态：已采纳
- 日期：2026-07-15

## 背景

微信支付、支付宝等真实在线支付依赖商家资质、正式凭据、回调域名、验签、对账和退款流程。项目无法替商家预置这些外部条件，也不能用消费者按钮伪装渠道支付成功。

## 决策

首个可运营版本只支持 `COD`（货到付款）和 `OFFLINE`（线下付款）。收款由商家后台确认并留痕；消费者不能直接修改支付状态。在线支付作为独立阶段实现，必须使用渠道沙箱验证支付单、验签、金额核对、幂等回调、查单、关单、对账和退款后再启用。

## 影响

- 当前版本适合接受货到付款或商家线下收款的试点订单，不宣称已接入在线支付。
- 接入任何支付渠道前必须新增支付订单模型和对应 ADR，渠道密钥只能通过受限 secret 提供。
- 在线支付不是首版上线阻塞项，但会限制面向陌生消费者的规模化交易。
