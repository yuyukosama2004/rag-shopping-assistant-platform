# ADR-0003：SKU 库存迁移策略

- 状态：已采纳
- 日期：2026-07-15

## 背景

历史商品使用商品级库存和 JSON 规格，无法可靠表示不同颜色、容量的售价与库存，也不利于订单快照和并发扣减。

## 决策

SKU 是售价、可售状态和库存的最小单位。旧商品迁移时为每个可售规格生成稳定 SKU；无法解析的记录进入失败表，不静默丢弃。订单项保存商品和 SKU 快照，库存变更写入流水，并以数据库条件更新和幂等机制保护不出现负库存或重复补偿。

## 影响

- 消费者下单必须提交 `skuId`，服务端重新读取售价和库存。
- 迁移脚本必须可重复执行，并在升级前通过加密备份提供版本级回滚能力。
- 旧商品级库存只用于迁移兼容，不再作为新交易的真实库存源。
