# 商家库存洞察页面

## 1. 范围

`INV-03` 在商家工作台新增 `/merchant/ai-operations`，导航名称为“AI 经营助手”。页面只读取
`INV-02` 提供的确定性库存洞察：

- 风险数量、总可用库存、近 30 天确认销量和无销量库存汇总。
- 风险卡片筛选、关键词查询、固定风险优先级排序和分页。
- 7/30/90 天确认销量、库存覆盖、销售额和计算时间。
- 单 SKU 脱敏证据抽屉，包括确认销售摘要、库存流水和数据限制。
- 加载、超时、权限不足、错误和空数据状态。
- 桌面表格与移动端卡片布局。

## 2. 安全边界

- 首次加载只调用列表和汇总 `GET` 接口，不调用大模型，不产生 AI 费用。
- 证据只在商家点击“查看证据”后按需读取。
- 页面没有自动改价、库存调整、商品上下架、订单、退款或自动清仓操作。
- 唯一业务跳转是进入现有库存管理页，由商家另行核对和操作。
- 证据页面不显示消费者姓名、手机号、地址、Token 或商家内部订单备注。
- 64 位 SKU ID 按字符串保存和传递，避免浏览器数字精度丢失；同时兼容历史负数测试 ID。

## 3. 展示口径

- 页面明确标注 `Asia/Shanghai` 和 7/30/90 天统计窗口。
- 风险同时显示中文文字、命中规则和后端规则编码，颜色不是唯一信息。
- `daysOfCover = null` 显示“近 30 天无确认销量”，不显示无解释的无穷大。
- 确认销售以收款时间为准；当前 SKU 销量未扣除退款数量。
- 不展示库存成本、利润、资金占用或具体降价底线。
- 页面提示“建议仅供经营决策参考，执行前请核对库存和成本”。

## 4. 自动化验证

前端组件测试覆盖：

- 初次加载只请求确定性列表和汇总，证据接口保持惰性。
- 风险文字、规则、7/30/90 天窗口和无销量库存覆盖说明。
- 风险筛选继续使用后端 `RISK_DESC` 稳定排序。
- 权限不足、空数据和移动端卡片结构。
- 证据抽屉只展示脱敏事实和后端限制说明。

验证命令：

```bash
cd biyesheji-frontend
npm ci
npm run test:unit
npm run build

```
