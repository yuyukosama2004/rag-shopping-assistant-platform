# 发布说明与已知限制

目标版本：`v0.8.0-alpha.2`
发布类型：单商家自托管预发布版
兼容经营方式：货到付款或线下确认（`COD/OFFLINE`）

## 1. 本阶段主要变化

- 完成店主初始化、店铺/员工、商品与 SKU、库存流水、订单履约、配送规则、退款申请和经营 AI 能力。
- 增加前端组件/API Mock、完整浏览器交易链路、并发库存、迁移、安全与故障恢复测试。
- 增加版本化部署、加密备份与隔离恢复、回滚、TLS 模板、结构化日志和监控告警。
- 移除单商家固定拓扑中不必要的 Nacos，网关改用 Docker 私有网络固定服务地址，降低内存占用和第三方攻击面。
- MySQL 固定为 8.0.46 Debian 基线并在构建时应用发行版安全更新；Redis 固定为 7.4.9 Alpine。
- CI 对源码、配置、业务镜像、基础设施镜像和监控镜像执行 High/Critical 门禁。
- 商品媒体支持本地持久卷和 S3/MinIO/R2 等兼容对象存储，外部 URL 仍保持 `/api/media/<id>`。
- S3 媒体已纳入加密备份/恢复流程，并补充 SKU 迁移校验与数据库回滚手册。
- Trivy 与 Gitleaks 使用固定版本和 SHA-256 校验安装；应用与 MySQL 运行镜像构建时应用系统安全更新。
- CI 已恢复为 GitHub 托管 `ubuntu-latest`，PR #54 和合并后的 `main` 均完成 13 项检查。

## 2. 升级注意事项

- 升级前必须创建并校验加密备份，保留上一版本应用镜像和前端归档。
- 首次从旧版升级会构建硬化 MySQL 镜像并重建数据库容器；数据卷保持不变，但必须预留维护窗口。
- Nacos 容器会作为已退役的孤儿服务被移除；旧 `nacos-data` 卷暂时保留，可在新版本稳定运行并完成备份后人工删除。
- Redis 会升级到固定版本；项目未依赖 Redis 持久化作为业务事实来源，MySQL 仍是订单与库存最终事实源。
- 升级后必须抽查登录、商品列表、购物车、下单、商家接单/发货、物流查看、监控目标和备份。
- 默认继续使用本地媒体卷。只有完成对象迁移并配置完整 S3 环境变量后才切换
  `MEDIA_STORAGE_TYPE=s3`，否则历史图片对象不会自动出现在新存储中。

## 3. 本版本验收结果

- GitHub Release 对应提交：`a7f8bb216367a3ba7476e2591d3b8ce3bfad6719`。
- PR #54 和 `main` 的后端、前端、配置、源码安全、镜像安全和完整 E2E 共 13 项检查全部通过。
- 目标服务器已从 `v0.8.0-alpha.1` 升级至 `v0.8.0-alpha.2`，四个应用容器均为 `healthy`。
- Flyway 最高迁移版本为 18；升级后抽查为 50 个商品、1044 个订单和 256 个用户。
- 加密备份与 SHA-256 校验通过，Prometheus 8 个 target 全部 `UP`，四服务未检出启动期
  ERROR/FATAL。
- 已实际回滚至 `v0.8.0-alpha.1` 并通过冒烟，再恢复 `v0.8.0-alpha.2` 并再次通过。

## 4. 已知限制

- 尚未完成真实商家试点；“商家无需开发人员介入即可经营”仍需现场证据。
- 尚未配置真实域名、公网 DNS 和 Let's Encrypt 证书，当前服务器只能视为内网/技术验收环境。
- Alertmanager 已验证本地接收闭环，但外部通知渠道取决于部署者配置。
- 不支持微信支付、支付宝或银行卡在线支付，也没有渠道对账；不得把 `COD/OFFLINE` 订单描述为已在线支付。
- 当前是单店部署，不提供平台管理员、多商家入驻、平台结算和跨商户数据隔离。
- AI 能力依赖外部模型服务和商家配置的凭据；模型不可用时不应阻塞商品、订单和履约主链路。
- S3 兼容实现已通过构建和自动化验证，但 AWS S3、MinIO、R2 等具体供应商的网络、权限、
  生命周期和恢复速度仍需部署者在自己的环境中验收。

## 5. 推荐升级窗口

1. 选择无待处理发货操作的低峰时段，通知商家停止写操作。
2. 记录当前 Tag、容器状态、磁盘余量和监控状态。
3. 执行加密备份，保留备份校验结果。
4. 升级并等待全部健康检查通过。
5. 执行核心业务抽查和监控/日志检查。
6. 失败时立即回退上一 Tag；成功后观察至少 30 分钟再结束窗口。

## 6. 稳定版门槛

只有在真实域名 HTTPS、外部告警接收和至少一家商家完整试点全部通过，且试点期间无 P0/P1 缺陷时，才考虑发布稳定版或 RC。当前版本继续使用 `alpha` 标识。
