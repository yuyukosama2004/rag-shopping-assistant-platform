# PhoneMall 压力测试演示指南

## 一、安装 JMeter

1. 下载 Java 17（如已安装跳过）: https://www.oracle.com/java/technologies/downloads/#jdk17-windows
2. 下载 JMeter: https://jmeter.apache.org/download_jmeter.cgi → 选 `Binaries > apache-jmeter-5.6.3.zip`
3. 解压到任意目录，进入 `bin/`，双击 `jmeter.bat` 启动 GUI

## 二、测试场景

模拟 1000 人同时抢购同一款手机（库存 100 件），验证：

- **不超卖**: 库存扣到 0 自动停止，不会出 -1
- **QPS 曲线**: 展示并发处理能力
- **响应时间**: 95% 请求在 xx ms 内完成
- **错误处理**: 库存不足时返回友好提示而非崩溃

## 三、准备工作（答辩前做好）

### 1. 确认手机上架且有库存

浏览器访问 `http://execute42.top`，挑一款手机，记下它的 ID（URL 如 `/product/1001`）

### 2. 重置库存为 100

```sql
-- 在服务器上执行
UPDATE t_stock SET total=100, locked=0, available=100, version=0 WHERE product_id=你的商品ID;
```

### 3. 准备 JWT Token

浏览器 F12 → Network → 登录后随便抓一个 `/api/` 请求 → 复制 Request Headers 里的 `Authorization: Bearer xxx...` 的值

## 四、创建 JMeter 测试计划

打开 JMeter，按以下步骤：

### 步骤 1: 添加线程组（模拟用户）

右键 "Test Plan" → Add → Threads → Thread Group:
```
Number of Threads: 100        （同时 100 个虚拟用户）
Ramp-up period: 10            （10秒内逐渐加压）
Loop Count: 10                （每个用户执行 10 次，共 1000 次请求）
```

### 步骤 2: 添加 HTTP Header Manager

右键 Thread Group → Add → Config Element → HTTP Header Manager:
```
Name: Authorization
Value: Bearer 你的JWT_TOKEN
```

再加一行:
```
Name: Content-Type
Value: application/json
```

### 步骤 3: 添加 HTTP Request（下单接口）

右键 Thread Group → Add → Sampler → HTTP Request:
```
Protocol: http
Server Name: execute42.top
Port: 80
Method: POST
Path: /api/order/orders
Body Data: {"items":[{"productId":你的商品ID,"quantity":1}]}
```

### 步骤 4: 添加监听器（图表）

右键 Thread Group → Add → Listener（以下各加一个）:

| 监听器 | 作用 | 给老师看什么 |
|--------|------|-------------|
| **View Results Tree** | 查看每个请求的响应详情 | 展示 "库存不足" 的正确响应 |
| **Summary Report** | 表格统计（QPS、错误率、平均响应时间） | 核心数据一览 |
| **Graph Results** | 实时折线图（吞吐量/响应时间） | 演示时最直观 |
| **Aggregate Graph** | 柱状图 | 报告截图 |

### 步骤 5: 保存测试计划

File → Save As → `PhoneMall_压测计划.jmx`

## 五、执行测试（演示流程）

### 建议演示顺序

1. **先展示架构图**（白板/PPT）
   ```
   用户请求 → Gateway → Sentinel限流 → Controller
   → Redis Lua 原子扣库存 → 成功 → RabbitMQ → 异步写MySQL
                           → 失败 → 返回"库存不足"
   ```

2. **展示数据库初始库存**: 浏览器打开后台或执行 `SELECT * FROM t_stock WHERE product_id=1001`

3. **启动 JMeter 压测**: 点绿色 ▶ 按钮，图表开始实时滚动

4. **边跑边解说**:
   - "现在 100 个用户在 10 秒内发起 1000 次下单请求"
   - 指 QPS 曲线："看，系统扛住了，平均 QPS 在 xx"
   - 响应时间在 xx ms 以内

5. **压测结束，展示结果**:
   - Summary Report 截图 → 0% 错误率（库存不足不算错误，是正确拒绝）
   - 回数据库查库存 → `available=0, locked=0`，没有超卖
   - "100 件库存，刚好卖了 100 件，没有多卖"

6. **拿着 JMeter 的 Aggregate Graph 截图**放入毕设论文

## 七、如果想生成 HTML 报告

压测完成后，命令行执行:
```bash
cd C:\path\to\jmeter\bin
jmeter -g PhoneMall_压测计划.jmx -o report
```
`report/index.html` 就是漂亮的测试报告，塞进论文。

## 八、常见问题

**Q: JMeter 跑不起来？**
A: 确认装了 JDK 17+，系统环境变量 `JAVA_HOME` 指向 JDK 目录。

**Q: 连接不上 execute42.top？**
A: 确认服务器 `online` 已开启，Windows 能 ping 通 execute42.top。

**Q: 压测数据不够震撼？**
A: 把 Thread Group 的 Number of Threads 改成 200，Loop Count 改成 20，库存设 500。
