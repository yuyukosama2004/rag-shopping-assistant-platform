# 单商家试点验收手册

本手册用于一家商家自部署、自运营的首轮试点。目标是验证商家能否独立完成经营闭环，而不只是确认系统能启动。

当前推荐试点基线为 `v0.8.0-alpha.2`。该版本已完成 GitHub 托管 CI、目标服务器升级、冒烟、
备份校验和实际版本回滚/恢复；这些结果只能作为试点前置条件，不能替代商家本人操作记录。

## 1. 试点前置条件

- 使用独立试点环境和可删除的数据，不直接导入唯一一份生产数据。
- 固定待验收版本 Tag，并记录部署时间、服务器配置和操作者。
- 当前使用 `v0.8.0-alpha.2` 时，确认活动 Release 的源提交为 `a7f8bb21`，四个应用镜像
  标签与活动 Release 一致；服务器仓库可包含该版本发布后的纯文档提交。
- 执行 `./start.sh backup`，确认生成加密备份且备份密钥另行保管。
- `./start.sh status` 中 MySQL、Redis、四个业务服务及监控服务均健康。
- 若面向公网，必须先完成真实域名 TLS 与仅开放 80/443 的检查。
- 当前支付方式限定为 `COD/OFFLINE`，不得录入真实线上支付凭据。

## 2. 商家独立操作任务

由商家本人操作，开发人员只观察和记录，不代替点击或填写。

1. 使用一次性初始化令牌创建店主账号并登录。
2. 完成店铺名称、说明、联系方式和基础配送规则配置。
3. 新建至少 5 个真实结构的商品，每个商品至少 2 个 SKU，并上传合法图片。
4. 设置库存、价格和上下架状态，确认消费者端展示一致。
5. 处理测试订单：查看详情、接单、填写承运商与运单号、发货。
6. 处理一次取消或退款申请流程，并核对订单与库存状态。
7. 查看监控状态、最近日志和备份结果，能找到故障排查入口。

## 3. 消费者任务

使用与商家不同的浏览器会话和账号完成：

1. 浏览、搜索并筛选商品，查看多 SKU 价格和库存。
2. 加入购物车，新增地址并提交 `COD/OFFLINE` 订单。
3. 重复点击提交不会生成无法解释的重复订单。
4. 查看订单状态变化、承运商和运单号。
5. 尝试输入脚本型文本或上传伪装文件，界面应安全拒绝或转义展示。

## 5. Go / No-Go 标准

满足以下全部条件才允许扩大试点：

- 商家无需开发人员代操作即可完成上架、接单和发货。
- 测试期间无丢单、越权、负库存、错误重复扣减或不可恢复数据错误。
- 所有 P0/P1 问题已修复并重新通过对应自动化与人工场景。
- 备份可读取，回滚目标和责任人明确。
- 公网试点已完成 TLS、端口、告警接收和续期演练。

出现任一数据一致性、安全、权限或无法回滚问题，结论立即为 No-Go，停止导入更多真实商品和用户。

## 6. 升级与回退

试点升级窗口内执行：

```bash
./start.sh backup
./start.sh upgrade <new-tag>
./start.sh status
```

健康或核心业务抽查失败时停止继续操作，并按记录的上一版本回退：

```bash
./start.sh rollback <previous-tag>
./start.sh status
```

回退后仍须保留失败版本日志、升级前备份和时间线，禁止为“恢复干净”而先删除证据。
